# Screenshots

The screenshots in this folder are used by [`README.md`](../../../README.md) (the top-level
project README). Every one of them is generated by
[`scripts/qa_popup_smoke.py`](../../../scripts/qa_popup_smoke.py) against real, on-screen
PrivacyFence UI — not mocked up by hand — so re-running the same command reproduces the same
image. See that script's own module docstring for the full explanation of what it does and why
(real click, real modal loop, real menu); this file just lists the exact invocation behind each
file below.

Requires the project's own venv (not a bare system `python3`) and, since it drives real clicks via
`System Events`, Accessibility permission granted to whatever terminal/IDE runs it (System
Settings → Privacy & Security → Accessibility). It pops real, visible windows/menus on your screen
for a couple of seconds each — run it locally, not headless, and don't touch the mouse/keyboard
while it runs.

| File | Command |
|---|---|
| `gmail-read-thread.png` | `.venv/bin/python scripts/qa_popup_smoke.py --scenario "gmail_get_thread" --screenshot-dir docs/images/screenshots --pause-seconds 3` |
| `sheets-write.png` | `.venv/bin/python scripts/qa_popup_smoke.py --scenario "drive_sheets_write_range" --screenshot-dir docs/images/screenshots --pause-seconds 3` |
| `menubar.png` | `.venv/bin/python scripts/qa_popup_smoke.py --scenario "Menu bar" --screenshot-dir docs/images/screenshots --pause-seconds 3` |
| `menubar-auto-accept-rules.png` | (same command as `menubar.png` — the menu-bar scenario produces both files in one run) |

`--scenario` matches by case-insensitive substring against the scenario name shown in the printed
report table, so each command above only pops the one relevant window/menu instead of sitting
through the full ~63-scenario suite. Each run writes its screenshot(s) under the name of the
scenario that produced them (see `_slugify()` in the script); rename the output to the filename
in the table above before committing it here.

`gmail-read-thread.png` and `sheets-write.png` are deliberately the two "kitchen sink" scenarios in
`qa_popup_smoke.py` — see the comments at those two scenarios in `_scenarios()` — so that these
two screenshots alone show every card a review-gate and a popup-gate dialog can carry
(AI-visibility checklist, PII/content-flag banner, seen-count, Claude's reason, ...), not just
their bare minimum shape.

Only the files actually referenced by `README.md` belong in this folder — it's a documentation
asset directory, not a general QA archive. If you regenerate a screenshot and no longer need the
old one, delete it rather than leaving it behind unused.
